# 🔧 404 错误排查指南

**问题：** 提交表单时报 404 错误

---

## 📋 可能原因

### 1. Railway 还未完成部署 ⏳

**检查方法：**
1. 访问 Railway 项目：https://railway.app
2. 查看 `deep-survey-backend` 项目的部署状态
3. 确认显示 "Deployed" 而不是 "Deploying"

**解决：** 等待 1-2 分钟，让 Railway 完成自动部署

---

### 2. 前端页面缓存 🔄

**问题：** 浏览器缓存了旧版本的前端代码

**解决方法：**

**方法 A：强制刷新**
- Windows/Linux: `Ctrl + Shift + R` 或 `Ctrl + F5`
- Mac: `Cmd + Shift + R`

**方法 B：清除缓存**
1. 打开浏览器开发者工具（F12）
2. 右键点击刷新按钮
3. 选择"清空缓存并硬性重新加载"

**方法 C：无痕模式**
- 打开浏览器的无痕/隐私模式
- 重新访问问卷页面

---

### 3. API 地址错误 🌐

**检查方法：**
1. 打开浏览器开发者工具（F12）
2. 切换到 "Console"（控制台）标签
3. 刷新页面，查看是否有以下日志：
   ```
   🔧 API 基础地址：https://xxx.railway.app/api
   🔧 当前域名：https://xxx.railway.app
   ```

**正常情况：** API 地址应该是 `https://你的域名.railway.app/api`

**异常情况：** 如果看到 `http://localhost:xxx` 或其他域名，说明部署有问题

---

### 4. 后端服务未启动 ❌

**检查方法：**
1. 访问健康检查接口：`https://你的域名.railway.app/health`
2. 正常响应应该是：
   ```json
   {
     "status": "ok",
     "timestamp": "2026-03-11T...",
     "feishuConfigured": true
   }
   ```

**异常响应：**
- 404：路由未配置
- 502/503：服务未启动
- 连接超时：域名错误

---

### 5. CORS 跨域问题 🔒

**检查方法：**
1. 打开浏览器开发者工具（F12）
2. 切换到 "Network"（网络）标签
3. 提交表单，查看请求详情
4. 检查是否有 CORS 错误

**解决：** 后端已配置 CORS，通常不是这个问题

---

## 🧪 测试步骤

### 步骤 1：检查后端服务

访问：`https://你的域名.railway.app/health`

✅ 正常：显示 JSON 数据  
❌ 异常：404/502/连接超时

---

### 步骤 2：检查前端页面

1. 打开问卷页面
2. 按 F12 打开开发者工具
3. 查看 Console 标签
4. 应该看到：
   ```
   🔧 API 基础地址：https://xxx.railway.app/api
   🔧 当前域名：https://xxx.railway.app
   ```

---

### 步骤 3：测试提交

1. 填写完整的表单（所有必填项）
2. 按 F12 打开开发者工具
3. 切换到 "Network"（网络）标签
4. 点击"提交申请"
5. 查看网络请求：
   - **请求 URL:** 应该是 `https://xxx.railway.app/api/submit`
   - **请求方法:** POST
   - **状态码:** 200（成功）或 400（验证失败）

---

### 步骤 4：查看详细错误

**如果报 404：**
1. 在 Network 标签点击失败的请求
2. 查看 "Response"（响应）标签
3. 截图并发给开发者

**如果报 400：**
- 这是正常的验证错误
- 说明后端服务正常，只是表单没填对

**如果报 500：**
- 服务器内部错误
- 查看 Console 标签的详细错误信息

---

## 📊 调试日志说明

提交时会输出以下日志：

```
🔧 API 基础地址：https://xxx.railway.app/api
🔧 当前域名：https://xxx.railway.app
📤 开始提交数据到：https://xxx.railway.app/api/submit
📝 提交数据：{...}
📝 完整 URL: https://xxx.railway.app/api/submit
📥 响应状态：200
📥 响应头：application/json
```

**如果看到 404 错误：**
```
❌ 响应错误：<!DOCTYPE html>...（HTML 内容）
❌ 错误 HTML 预览：<!DOCTYPE html><html>...
```

这说明请求到了错误的地址（不是后端 API）

---

## ✅ 成功标志

提交成功后应该看到：

1. **Network 标签：** 状态码 200
2. **Console 标签：** 
   ```
   提交成功：{success: true, recordId: "..."}
   ```
3. **页面显示：** "申请提交成功！"绿色提示
