## 3. 填写问卷

### 3.1 完整填写流程

**第 1 步：基本信息**
1. 姓名：输入真实姓名（必填）
2. 部门/单位：输入工作单位（必填）
3. 邮箱：输入常用邮箱（必填）
4. 微信/手机号：选填

**第 2 步：大模型使用经验**
1. 使用经验：单选（必填）
   - 没用过
   - 偶尔使用（每周 1-2 次）
   - 经常使用（每天）
   - 深度用户（用于工作/开发）

2. 使用场景：多选（必填，至少选 1 个）
   - 日常问答
   - 文档写作
   - 代码开发
   - 数据分析
   - 图像生成
   - 其他

3. 项目经验：单选（必填）
   - 没有
   - 尝试过但未完成
   - 完成过 1-2 个小项目
   - 完成过多个项目

**第 3 步：参与意愿**
1. 参与动机：填写原因（必填，至少 20 字）
   - 示例："希望能与同行深度交流大模型技术，分享实战经验，共同成长进步"

2. 时间投入：单选（必填）
   - 偶尔参与
   - 每周能参与 1-2 次讨论
   - 每周能参与 3 次以上
   - 可以深度参与和组织活动

3. 能分享的内容：多选（选填）
   - 技术方案分享
   - 实战案例分享
   - 行业洞察
   - 资源对接
   - 活动组织
   - 其他

**第 4 步：期望与建议**
1. 期望：选填
2. 其他建议：选填

**第 5 步：提交**
- 点击"提交申请"按钮
- 等待提交完成
- 显示成功页面

### 3.2 填写技巧

**进度条说明：**
- 页面顶部有进度条
- 实时显示填写进度（0% - 100%）
- 建议达到 100% 后再提交

**字数统计：**
- 参与动机字段有字数统计
- < 20 字：红色警告
- ≥ 20 字：绿色正常
- 建议认真填写，这是重要筛选依据

**必填项标记：**
- 带 `*` 的是必填项
- 未填必填项无法提交
- 系统会提示具体哪个字段未填

### 4.2 浏览器测试工具

**Chrome 开发者工具（推荐）**

**打开方式：**
- Windows：`F12` 或 `Ctrl+Shift+I`
- Mac：`Cmd+Option+I`
- 右键 → 检查

**查看控制台：**
1. 打开开发者工具
2. 点击"Console"（控制台）标签
3. 查看是否有错误信息

**正常情况：**
```
开始提交数据到：https://tender-caution-atm-modules.trycloudflare.com/api/submit
提交数据：{姓名："田福成", ...}
响应状态：200
提交成功：{success: true, message: "提交成功", id: 1}
```

**错误情况：**
```
❌ Failed to load resource: net::ERR_NAME_NOT_RESOLVED
   → 域名解析失败，检查网络连接

❌ Failed to fetch
   → 无法连接到服务器，检查网络或服务器状态

❌ AbortError: The operation was aborted due to timeout
   → 提交超时，网络太慢或服务器无响应
```

**查看网络请求：**
1. 打开开发者工具
2. 点击"Network"（网络）标签
3. 填写表单并提交
4. 查看 `submit` 请求

**正常请求：**
```

Request Method: POST
Status Code: 200 OK

Request Payload:
{
  "姓名": "田福成",
  "部门单位": "技术部",

  "success": true,
```

## 5. 常见问题

### 5.1 访问问题

**问题 1：无法打开网站**

**现象：**
- 浏览器显示"无法访问此网站"
- 或显示"连接超时"

**排查步骤：**
1. 检查网络连接
   ```javascript
   console.log('网络状态:', navigator.onLine);
   // 应该显示 true
   ```

2. 检查网址是否正确
   - 确认链接完整
   - 没有多余空格
   - https 开头

3. 更换浏览器
   - 尝试 Chrome/Firefox/Safari
   - 避免使用老旧浏览器

4. 检查防火墙
   - 公司网络可能屏蔽
   - 尝试手机 4G/5G 网络

**解决方案：**
- ✅ 连接稳定网络
- ✅ 使用推荐浏览器
- ✅ 联系管理员获取新链接

---

**问题 2：页面加载慢**

**现象：**
- 页面加载超过 10 秒
- 或一直显示"加载中..."

**排查步骤：**
1. 测试网速
   - 访问 speedtest.net
   - 检查下载速度

2. 清除浏览器缓存
   - Chrome：`Ctrl+Shift+Delete`
   - 选择"缓存的图像和文件"
   - 清除数据

3. 检查服务器状态
   - 联系管理员确认服务器运行

**解决方案：**
- ✅ 切换到更快的网络
- ✅ 清除浏览器缓存
- ✅ 稍后重试

---

### 5.2 提交问题

**问题 3：提示"网络错误"**

**现象：**
```
提交失败：网络错误，请检查网络连接
```

**排查步骤：**
1. 检查网络状态
   ```javascript
   console.log('网络状态:', navigator.onLine);
   ```

2. 查看控制台错误
   - 按 `F12` 打开开发者工具
   - 查看 Console 标签
   - 记录错误信息

3. 测试 API 连接
   ```javascript
   fetch(window.location.origin + '/health')
     .then(r => r.json())
     .then(d => console.log('服务器状态:', d));
   ```

**可能原因：**
- ❌ 网络连接不稳定
- ❌ Cloudflare 隧道临时故障
- ❌ 服务器未运行

**解决方案：**
- ✅ 检查网络连接
- ✅ 刷新页面重试
- ✅ 联系管理员获取新链接

---

## 6. 管理员操作

### 6.1 查看提交数据

**方式 1：服务器日志**
```bash
# 查看最新提交
tail -f ~/.openclaw/workspace/deep-survey-backend/server.log

# 查看提交统计
grep "✅ 新提交" ~/.openclaw/workspace/deep-survey-backend/server.log | wc -l
```

**方式 2：API 接口**
```bash
# 获取所有提交数据
curl -H "Authorization: Bearer your-secret-key" \
     http://localhost:3000/api/submissions
```

**响应示例：**
```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "姓名": "田福成",
      "邮箱": "tianfch@example.com",
      "使用经验": "经常使用（每天）",
      ...
      "submittedAt": "2026-03-08T07:27:22.391Z"
    }
  ]
}
```

### 6.2 导出数据

**导出 CSV：**
```bash
curl http://localhost:3000/api/export \
  -o submissions.csv
```

**打开文件：**
- Windows：双击 `submissions.csv`
- Mac：用 Numbers 或 Excel 打开

### 6.3 服务器管理

**启动服务器：**
```bash
cd ~/.openclaw/workspace/deep-survey-backend
npm start
```

**停止服务器：**
```bash
# 找到进程 ID
ps aux | grep "node server.js"

# 杀死进程
kill <PID>
```

**重启服务器：**
```bash
npm restart
```

**查看状态：**
```bash
# 检查端口
lsof -i :3000

# 查看日志
tail -f server.log
```

### 6.4 部署到 Railway（永久链接）

**步骤 1：安装 Railway CLI**
```bash
npm install -g @railway/cli
```

**步骤 2：登录**
```bash
railway login
```

**步骤 3：初始化项目**
```bash
cd ~/.openclaw/workspace/deep-survey-backend
railway init
```

**步骤 4：设置环境变量**
```bash
railway variables set PORT=3000
railway variables set API_SECRET=your-secret-key
```

**步骤 5：部署**
```bash
railway up
```

**步骤 6：绑定域名**
```bash
railway domain
```

**获得永久链接：**
```
https://deep-survey-production.up.railway.app
```
