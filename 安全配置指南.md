# 🔐 安全配置指南

**重要：** 本文档包含敏感配置信息，请勿提交到 Git！

---

## ⚠️ 环境变量配置（必须）

### Railway 环境变量

请登录 Railway 后台配置以下变量：

**飞书应用配置：**
```env
FEISHU_APP_ID=your-feishu-app-id-here
FEISHU_APP_SECRET=your-feishu-app-secret-here
FEISHU_APP_TOKEN=your-feishu-app-token-here
FEISHU_TABLE_ID=your-feishu-table-id-here
```

**服务配置：**
```env
PORT=3000
NODE_ENV=production
```

---

## 📋 配置步骤

### 1. 获取飞书应用凭证

1. 访问 https://open.feishu.cn/app
2. 创建或选择应用
3. 进入"凭证管理"
4. 记录 App ID 和 App Secret

### 2. 创建飞书多维表格

1. 访问 https://feishu.cn
2. 创建新的多维表格
3. 配置所需字段
4. 从 URL 获取 app_token 和 table_id

格式：`https://feishu.cn/base/{app_token}/table/{table_id}`

### 3. 配置 Railway

1. 登录 https://railway.app
2. 选择项目
3. 点击 "Variables" 标签
4. 添加上述环境变量
5. 保存后自动重新部署

---

## ✅ 验证配置

部署完成后，访问健康检查接口：

```
https://你的项目.up.railway.app/health
```

**正常响应：**
```json
{
  "status": "ok",
  "timestamp": "2026-03-11T...",
  "feishuConfigured": true
}
```

如果 `feishuConfigured: false`，说明环境变量未正确配置。

---

## 🔒 安全提醒

1. **不要硬编码密钥** - 始终使用环境变量
2. **不要提交 .env 文件** - 已在 .gitignore 中排除
3. **定期轮换密钥** - 建议每 3-6 个月
4. **限制 CORS 域名** - 仅允许信任的域名访问

---

## 🚨 密钥泄露处理

如果密钥不慎泄露：

1. **立即撤销** - 在飞书/Railway/GitHub 撤销密钥
2. **重新生成** - 生成新的密钥
3. **更新配置** - 在 Railway 更新环境变量
4. **清理历史** - 从 Git 历史中清理泄露的密钥
